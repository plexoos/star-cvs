How to create your own custom filter:

  1. Create a local directory StRoot if you have not made it yet.
  2. Create there your own filter class derived from "StFilterABC" abstact interface defined within "StRoot/StEventUtilities/StEventHelper.h". as follows:
    • mkdir StRoot
    • cd StRoot
    • cvs co -d StCustomFilter StRoot/StEventUtilities/examples

    One can create as many different filter classes as needed in one directory if he/she likes.
  3. Apply "cons" to create the share library 
    cons +StCustomFilter
  4. Customize "doEvents.C" macro:
    • load you share library:

      Find the line:
      gSystem->Load("StEventDisplayMaker");
      and add your own one after that:
      gSystem->Load("MyCustomFilter");

    • Find the lines: 
      displayMk->AddFilter(new StFilterDef("MainFilter"));
displayMk->AddFilter(new StMuDstFilterHelper("MuL3Filter",kFALSE));
    • remove it or add your own custom filter('s) 
      displayMk->AddFilter(new StCustomFilter("MyOwnOne",kFALSE));
      where:
      1. the first parameter is your filter name that will be used as the filter label to be displayed;
      2. the second parameters is the activation flag to mark whether your filter will be active (i.e. whether it will be applied to make the cut);
      The order is important because the filters are applied in the order you put them in your macro. I.e. in this particular case the "MainFilter" is to be applied first, then all events those passed the first filter are to be tested by the second one if it is active etc.
  5. Launch your copy of "doEvent.C" providing the last parameters is "display"
