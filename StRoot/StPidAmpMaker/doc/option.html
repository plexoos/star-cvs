Fit options:


"I"------fit individual de/dx~p/z band. If the ChannelCollection is a default ChannelCollection, a file "ChannelCollectionName+Bands.root" will be produced.
"B"------fit dedx~beta*gamma instead of fit individual de/dx~p/z band. this will cause one set of fitted band parameters for all particle types. A file "ChannelCollectionName+BGBands.root" will be produced.
"A"------fit amplitude. A file "ChannelCollectionName+Amp.root" will be produced.
"R"------fit resolution.
"T"------iterate de/dx~beta*gamma fitting twice. if "B" is not slected, this option will be ignored.

A ChannelCollectionName is sth like"nhitsBin_0_10_20_45_ptBin_0_Infinity_", it depends on what ChannelCollection the user have added into the StPidAmpManager.

Please note:
1. Amplitude fitting or resolution fitting won't make sense if the band does not get fitted first. So if "A" or "R" is selected, either "I" or "B" should be selected. If the user forget to do so, the code will pick up "B".
2. Only the "*BGBand.root" file contains the calibration constant.

Draw options:


"I"------draw individual band fitting (dedx~p/z for each particle type).
"B"------draw de/dx~beta*gamma fitting. If no "B" in the fit option, this option will be ignored.
"A"------draw amplitude fitting.
"R"------draw resolution fitting.
