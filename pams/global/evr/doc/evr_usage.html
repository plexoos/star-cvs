EVR - Usage

The package is driven by a parameter table, evr_evrpar. Here is the meaning of those variables. The user should feel free to experiment with them:

  • minhit: minimum number of hits on a track used to fit the vertex.
  • cutxy: First pass cut in x-y plane for outlier removal. Track should point back to the vertex at most this distance (in cm).
  • cutz: The same as cutxy but for the beam axis. This is usually more relaxed.
  • cut2: Second pass cut for outlier removal. Common value for x-y and z.
  • cut3: Last pass cut value. This is usually a fraction of a cm.
  • ptmin: Minimum pt track need in order to be used in the fit.
  • match and use: These control the kind of tracks to be used. TPC, SVT or global. See Table below for combinations. Numbers are positive integers. x means "do not care". The use of match=1 is not recomended.
use match
0 1 matched svt tracks. Use tpc number of tracks. Do not use.
1 1 Same as above but use svt number of tracks. Do not use.
0 0 tpc tracks are used to fit vertex
1 0 svt tracks are used to fit vertex
x >1 invalid entry
2 x global tracks are used to fit vertex
>2 x invalid entry
